# Implementation Plan: Comprehensive Stem×Parameter Sweep Matrix

**Branch**: `015-sweep-matrix` | **Date**: 2026-03-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/015-sweep-matrix/spec.md`

## Summary

Replace the current limited sweep system (capped at 3 stems, minimal permutations) with a comprehensive matrix sweep engine. Add 13 new Vamp algorithm wrappers. Run every algorithm against all affinity-matched stems with full parameter permutations on a representative 30-second audio segment. Store results in a unified report (metadata) + per-algorithm files (full data). Provide CLI ranking (`sweep-results`) and review UI comparison view. Auto-select and export best results.

## Technical Context

**Language/Version**: Python 3.11+
**Primary Dependencies**: vamp, librosa, numpy, click 8+, Flask 3+, rich (progress display), concurrent.futures (stdlib)
**Storage**: JSON files (sweep reports, per-algorithm results), TOML (sweep config)
**Testing**: pytest
**Target Platform**: macOS (primary), Linux
**Project Type**: CLI tool + local web UI
**Performance Goals**: Full sweep matrix on 6-stem, 3.5-min song in <5 minutes (30s sample segment)
**Constraints**: Vamp/madmom algorithms run in .venv-vamp subprocess for ABI isolation
**Scale/Scope**: ~35 algorithms × ~5 stems × ~4 params = ~700 permutations max; typical run ~200-300
